# Решение проблемы с установкой зависимостей

## Проблема
Ошибка `EACCES: permission denied` при установке npm пакетов связана с правами доступа к кэшу npm.

## Решения (попробуйте по порядку):

### Решение 1: Очистка кэша npm и исправление прав
```bash
# Удалите проблемный файл из кэша
rm -rf ~/.npm/_cacache/content-v2/sha512/84/5b/fa5203c56e14fcd13d2178f016a3e9bd2598a3849dd724c0e43eed72c6211b32376a1175af74f237830f55f56b77030d05069ed983820c34da16ba9b2bd7

# Очистите временные файлы
rm -rf ~/.npm/_cacache/tmp/*

# Попробуйте установить снова
npm install
```

### Решение 2: Исправление прав доступа к кэшу npm
```bash
# Исправьте права доступа к директории кэша
sudo chown -R $(whoami) ~/.npm

# Затем попробуйте установить
npm install
```

### Решение 3: Использование другого менеджера пакетов
Если npm продолжает вызывать проблемы, установите yarn:
```bash
# Установка yarn через npm (если npm работает)
npm install -g yarn

# Или через Homebrew
brew install yarn

# Затем используйте yarn для установки зависимостей
yarn install
```

### Решение 4: Использование nvm (рекомендуется)
Если у вас проблемы с глобальным npm, используйте nvm для управления версиями Node.js:
```bash
# Установите nvm (если еще не установлен)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Перезагрузите терминал или выполните:
source ~/.zshrc

# Установите последнюю LTS версию Node.js
nvm install --lts
nvm use --lts

# Теперь npm должен работать корректно
npm install
```

### Решение 5: Переустановка npm
Если ничего не помогает, переустановите npm:
```bash
# Удалите текущий npm
sudo rm -rf /usr/local/lib/node_modules/npm

# Переустановите через Homebrew
brew install node

# Или через официальный установщик с nodejs.org
```

## После решения проблемы

После того как установка пройдет успешно, запустите проект:
```bash
npm run dev
```

Проект будет доступен по адресу: http://localhost:3000
